作業環境の準備

前提条件: D:\CIDLS が CIDLS リポジトリルート。D:\production\.venv に Python 3.11 仮想環境。devrag.exe が %LOCALAPPDATA%\devrag\devrag.exe に存在すること。

環境変数の設定 (セッション開始時)

PowerShell セッション変数
$BIN = "$env:LOCALAPPDATA\devrag\devrag.exe" $CFG = "D:\CIDLS\devrag-config.json" $PY = "D:\production\.venv\Scripts\python.exe" cd D:\CIDLS
以降のコマンドはこれらの変数を前提とする

環境確認コマンド

devrag 動作確認
& $BIN search --top-k 2 --config $CFG "CAPDkA"
count=3 以上・similarity=0.90 以上が返れば正常
Python 環境確認
& $PY --version
Python 3.11.x が表示されれば正常
索引件数確認
& $BIN list --config $CFG
count=38 前後が正常。0 ならインデックス再構築が必要

devrag 操作手順

[禁止] devrag index-code *.py は使用禁止。 Python ファイルでゼロベクトルが発生し、ベクトル検索が全件 count=0 になる。

クリーン再インデックス (DB 破損時・定期メンテ)

Step 1: DB 削除
Remove-Item D:\CIDLS\.devrag\vectors.db -Force
Step 2: AGENTS.md 索引
& $BIN index D:\CIDLS\AGENTS.md --config $CFG
283 チャンク。"Successfully indexed" が表示されれば OK
Step 3: documents/*.md 一括索引
Get-ChildItem D:\CIDLS\documents -Filter "*.md" | ForEach-Object { & $BIN index $_.FullName --config $CFG 2>&1 | Where-Object { $_ -match "Successfully" } }
Step 4: .github/**/*.md 一括索引
Get-ChildItem D:\CIDLS\.github -Recurse -Filter "*.md" | ForEach-Object { & $BIN index $_.FullName --config $CFG 2>&1 | Where-Object { $_ -match "Successfully" } }
Step 5: 索引確認
& $BIN list --config $CFG
count=38 前後が正常
Step 6: 検索テスト
& $BIN search --top-k 3 --config $CFG "CAPDkA cycle" # 期待: count=3, similarity >= 0.90

新規 md ファイル追加時

単一ファイル追加索引
& $BIN index D:\CIDLS\documents\新規ファイル.md --config $CFG
既存 DB を消さずに追記。ただし index-code は使わないこと。

注意事項

項目正しい方法間違い (禁止)
引数順序 search --top-k 5 --config $CFG "クエリ" search "クエリ" --config $CFG --top-k 5
Python 索引 使用しない index-code *.py (ゼロベクトル汚染)
設定ファイル devrag-config.json (--config 明示) config.json (旧設定、参照不可)
code_patterns キー devrag-config.json に含めない 記述すると警告・動作不安定

トラブルシューティング

devrag 検索が count=0 を返す

根本原因: index-code (Python ファイル) でゼロベクトルが混入し、ベクトル検索が破壊される。
確認項目コマンド期待値
DB 存在確認 Test-Path D:\CIDLS\.devrag\vectors.db True
索引件数確認 & $BIN list --config $CFG count=38 前後
単語検索テスト & $BIN search --top-k 2 --config $CFG "CAPDkA" count ≥ 3

解決手順: 上記「クリーン再インデックス (02 devrag 操作)」を実施する。

generate_*.py が失敗する

症状原因対処
ModuleNotFoundError 仮想環境が有効化されていない $PY = "D:\production\.venv\Scripts\python.exe" を明示して実行
FileNotFoundError CWD が D:\CIDLS でない cd D:\CIDLS してから実行
openpyxl 未インストール 依存ライブラリなし uv add openpyxl (pip 禁止)

Unicode エラー (cp932)

症状: UnicodeEncodeError: 'cp932' codec can't encode character

原因: コード・バッチ・ログに Unicode 絵文字 (絵文字系文字) が含まれている。
対処: 絵文字を削除し ASCII 文字のみ使用する。バッチファイルは chcp 932 を先頭に追加する。

インシデント対応 SLA

レベル種別対応期限内容
Level 1 軽微なバグ 72時間以内 パッチ適用
Level 2 機能停止・流出疑い 4時間以内 調査開始 + 関係者通知
Level 3 情報漏洩確定 1時間以内 サービス停止検討 + 72時間以内当局報告
