はじめに

2.1 CIDLSプラットフォームとは

CIDLSプラットフォームは、コンセプト入力からアプリ生成・ドキュメント一式生成・自律進化保守ループまでを一気通貫で実行するAI駆動開発基盤です。AGENTS.md（全ルールのSoT）を中心に、日次で自律進化しながら商用品質の成果物を継続的に生成します。

2.2 基本的な使い方の流れ

Step 1: index.html を開く

D:\CIDLS\index.html をブラウザで開きます。全成果物へのクイックアクセスナビゲーションが表示されます。

Step 2: 目的の成果物を選ぶ

カードをクリックして対象ドキュメントまたはアプリにアクセスします。シームレス生成ドキュメント5種、kanban、マインドマップなどが即座に開けます。

Step 3: タブで情報を確認・操作

各HTMLドキュメントは10タブ構成です。目的に応じたタブをクリックして情報を確認します。

Step 4: 改善・追加が必要な場合

VSCode Copilot AgentにAGENTS.mdのルールを参照させながら指示を出します。全操作はCAPDkAサイクルに従って実行されます。

kanban操作

7.1 kanbanのタスク管理フロー

タスク追加（Backlog）

新規タスクは scripts\generate_project_kanban.py のテンプレートに追加します。タスクIDはDA-NNN形式で付番します。

タスク進捗更新

CAPDkAサイクルの進行に合わせてタスク状態を Backlog → Todo → InProgress → Review → Done に更新します。

Done判定条件

「no issues自己申告だけでDone禁止」（AGENTS.md [Q4.3]）。客観データ + 第三者レビュー + ユーザー検証が揃った時点でDoneに移動します。

7.2 kanban更新時の水平展開同期

必須: CAPDkA完了時に同時更新

kanban更新時は同時に以下も確認・更新してください:
- index.html（新規ファイル追加時のみ）
- cidls_platform_overview.html（成果物追加時のみ）
- MO必須出力4成果物チェックリスト（毎回）

トラブルシューティング

9.1 よくある問題と解決方法

問題: HTMLがブラウザで正常に開かない

確認: ファイルパスに日本語が含まれている場合、ブラウザが対応しているか確認する。
解決: Chrome または Edge で開く。必要であれば http-server などのローカルサーバーを使用する。

問題: Pythonスクリプトが実行できない

確認: D:\production\.venv が存在するか確認する。
解決: 「構成定義書兼運用手順書」のセットアップタブを参照して仮想環境を再構築する。

問題: devrag検索が失敗する / 件数が少ない

確認: %LOCALAPPDATA%\devrag\devrag.exe の存在確認。
解決: setup_devrag.ps1 を再実行する。index-code は実行禁止。

問題: 日本語ファイル名のHTMLが Windows Task Scheduler で処理できない

確認: run_daily_self_evolution.cmd の文字コードが ANSI (Shift-JIS) であることを確認する。
解決: cmd ファイルを Shift-JIS で保存し直す。

9.2 エラーメッセージ対応表

エラー原因対処
UnicodeEncodeError: cp932コードまたはファイル内にUnicode絵文字が含まれる絵文字を全て削除する（AGENTS.md [TS.1]）
ModuleNotFoundError仮想環境のパッケージ不足uv pip install で対象パッケージを追加
FileNotFoundError: devrag.exedevragバイナリが未インストールsetup_devrag.ps1 を実行する

FAQ

Q. index.html から全てのドキュメントに一発でアクセスできますか？
A. はい。index.html の Quick Access セクションには全主要ドキュメントへのカードが配置されています。目的のカードをクリックすることで3クリック以内で各ドキュメントにアクセスできます。
Q. 日次自律進化ループが実行されたか確認するには？
A. D:\CIDLS\logs\daily_self_evolution\ フォルダを開き、当日付のログファイルが存在することを確認します。ログ内に ERROR 文字列がないことも確認してください。
Q. 新しいドキュメントを追加する際に何を同期すればよいですか？
A. AGENTS.md [HS] 水平展開同期に従い、以下を同期してください: index.html（カード追加）/ cidls_platform_overview.html（成果物リスト更新）/ generate_cidls_platform_overview.py（ARTIFACT_FILES追加）/ ドキュメント一覧.html（カード追加）/ project_kanban.html（タスク記録）
Q. Unicode絵文字はなぜ使用禁止なのですか？
A. Windows環境のcp932エンコーディングでUnicode絵文字を処理するとUnicodeEncodeErrorが発生します。AGENTS.md [G1] により全成果物で禁止されています。代替としてASCII文字（[OK], [ERROR], [INFO]等）を使用してください。
Q. AGENTS.md を変更する際の注意点は？
A. AGENTS.md変更前に必ずAGENTS.md.bakを作成してバックアップを取ってください。変更後は[DIFF]差分報告フォーマットで変更内容を記録し、水平展開同期を実施してください。
