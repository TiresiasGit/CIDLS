バイブコーディングの罠 -- AI時代の3大負債

AIがコードを書ける時代、「雰囲気で生成させる」だけでは 前例なき規模の技術的負債が蓄積します (AV.0)。

AV.8 技術負債

設計妥協によるコスト増大

AIコードのバグ密度は人間の1.7倍、セキュリティ脆弱性は2.74倍 (CodeRabbit/Veracode 2024調査)。ノーチェックコミットで本番障害が発生します。

AV.8 理解負債

誰も理解できないブラックボックス化

AI生成コードを丸ごとコミットし続けると、修正時に誰も コードを理解できない「ポチョムキン状態」(AV.7) に陥ります。

AV.8 認知負債

AI全依存による開発力の空洞化

設計・判断・レビューをAIに委ねすぎると、担当者退職時に システム全体がブラックボックス化します (DS.3 組織知断絶)。

AV.9 4大罠

ノーチェックコミット / プロンプト依存症

レビューなしコミット、ブラックボックス建築、プロンプト依存、 セキュリティ無関心 -- これらが同時に進行し、チームを崩壊させます。

FAQ

どのAIサービスに対応していますか?
GitHub Copilot (Visual Studio Code) を主要ターゲットとして設計しています。 AGENTS.md / .github/copilot-instructions.md は Copilot に自動読み込みされます。 ChatGPT / Claude / Gemini CLI にも対応しています。 LLM API Key はお客様自身のアカウントをご利用いただきます (DISTSEC.5: 運営者 API Key を配布物に含めません)。
Windows以外のOSでも動作しますか?
現在のプロンプトキットはWindows 10/11向けに最適化されています (T1.1)。 WSL禁止 / PowerShell推奨の設計です。 macOS / Linux への対応は方法2 ToB導入支援の個別調整で対応可能です。
Secret やAPIキーは配布物に含まれますか?
含まれません。DISTSEC.2 絶対禁止事項に従い、 配布ZIPに.env / 秘密鍵 / API Key / Webhook Secret は一切含まれません。 LLM API KeyはSECRET品質ゲート (DISTSEC.6) で自動検査済みです。
CAPDkAサイクルとは何ですか?
PDCAではなくC(Check現状把握) -> A(ActionPlan計画) -> P(Plan) -> D(Do実装) -> kA(Knowledge蓄積) の順で実行する永久循環型開発サイクルです (P2.1)。 kA完了後は即座にCへ回帰し、品質・知識・精度が螺旋的に向上します。 日次11:00の自動実行がこのサイクルを自律的に駆動します。
テストカバレッジの目標はありますか?
ユニットテストカバレッジ >= 90% が目標です (Q1.2)。 現在のCIDLS自体はテストカバレッジ 0% が残課題 (F16) として管理されており、 pytest導入を次サイクルの即時アクションとして計画中です (DA-021)。 「完璧な状態でないものを完璧と主張しない」AGENTS.md Q1.3 に従い正直に開示しています。
SaaS化や共有リンク機能は提供されますか?
現在の提供方式は DISTSEC.3 方法1 (プロンプトキット) および 方法2 (ToB導入支援) です。Web公開SaaS (方法5) は DISTSEC.3 Web SaaS禁止条件 (認証/課金/監査ログ/OWASP等) を 全件クリアするまで提供しません。これはセキュリティ上の判断です。
