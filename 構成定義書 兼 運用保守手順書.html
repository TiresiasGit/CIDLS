サービス構成図

2.1 論理構成

[Layer 1] 入力層

コンセプト入力フォーム / チャットUI / ファイルアップロード

index.html (クイックアクセス) / シームレス生成ドキュメント5種 / LP.html (商用)
エンドユーザー・顧客が最終成果物にアクセスする層

運用手順

4.1 日次自律進化ループ運用手順

Step 1: 実行前確認

D:\CIDLS\logs\daily_self_evolution\ に前日のログが存在することを確認する。エラーがあれば障害対応タブを参照する。

Step 2: 自動実行 (11:00)

Windows Task Scheduler が scripts\run_daily_self_evolution.cmd を自動実行する。手動実行が必要な場合は下記コマンドを使用する。

cd D:\CIDLS scripts\run_daily_self_evolution.cmd

Step 3: 実行結果確認

logs\daily_self_evolution\ に当日付のログファイルが作成されていることを確認する。エラーがないことを確認する。

Step 4: kanban更新確認

project_kanban.html を開き、タスク状態が正しく更新されていることを確認する。

4.2 kanban定期更新手順

CAPDkAサイクル完了時

scripts\generate_project_kanban.py を編集し、回数・タスク内容・完了事項を更新する。その後、手動でproject_kanban.htmlに反映する（または生成スクリプトを実行する）。

4.3 新規成果物追加時の水平展開同期

必須: [HS]水平展開同期

新規HTMLファイル追加時は以下の全箇所を同期すること（省略=G1違反）:
1. index.html — Quick Accessカード追加
2. cidls_platform_overview.html — 成果物リスト更新
3. scripts/generate_cidls_platform_overview.py — ARTIFACT_FILES追加
4. ドキュメント一覧.html — カード追加
5. project_kanban.html — 第N回完了タスク記録

リカバリ手順

7.1 ロールバック手順

前提: [DIFF]差分報告が必要

全変更に対してBefore/After・変更理由・ロールバック手順を記録すること（AGENTS.md [DIFF.3]）。

HTML成果物のロールバック

git revert または バックアップからの復元。変更前ファイルが.bak存在する場合はリネームで復元。

生成スクリプトのロールバック

git reset --hard または バックアップからの復元。必ず動作確認後に本番適用する。

devragインデックスのロールバック

.devrag/vectors.db を削除後、index コマンドで再構築する。

7.2 AGENTS.mdのロールバック

重要: AGENTS.md変更は慎重に

AGENTS.md はSoTであるため、変更前にAGENTS.md.bakを必ず作成すること。ロールバック時は .bak をリネームして復元する。

# バックアップ作成 copy D:\CIDLS\AGENTS.md D:\CIDLS\AGENTS.md.bak # ロールバック copy D:\CIDLS\AGENTS.md.bak D:\CIDLS\AGENTS.md

7.3 仮想環境の再構築

venv破損時

.venv フォルダを削除後、セットアップ手順Step 2を再実行する。

cd D:\production rmdir /s /q .venv uv venv .venv uv pip install openpyxl duckdb pandas pyarrow

10.2 よくある障害と対処

日次進化ループが実行されない

確認: Task Schedulerのタスク「CIDLS_Daily_Self_Evolution_1100」の状態を確認。
対処: タスクが無効化されている場合は有効化する。register_daily_self_evolution_task.ps1 を再実行してタスクを再登録する。

devragインデックス検索が失敗する

確認: %LOCALAPPDATA%\devrag\devrag.exe の存在確認 / devrag-config.json の内容確認。
対処: setup_devrag.ps1 を再実行してセットアップをやり直す。index-code は実行禁止。

Excel生成スクリプトでUnicodeEncodeError

確認: コード内にUnicode絵文字が含まれていないか確認。
対処: 絵文字を全て削除し、ASCII文字列に置き換える（AGENTS.md [TS.1]準拠）。

Python仮想環境が見つからない

確認: D:\production\.venv の存在確認。
対処: リカバリ手順タブの「仮想環境の再構築」を実行する。

n8n が停止して外部発報が届かない

確認: docker ps で n8n コンテナの状態を確認。localhost:5678 の応答を確認。
対処: docker compose -f docker-compose.n8n.yml restart n8n を実行する。
応急: n8n停止時はアプリ側は [WARNING] ログを出力して処理を継続する設計 (フェイルクローズ)。復旧後に再送は行わないため、手動での発報再実行を検討する。
参照: AGENTS.md [N8N.2]
