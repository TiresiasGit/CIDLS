1. 要求要件定義

1.1 背景・目的

CIDLSは「コンセプト入力からアプリ出力・自律進化まで一気通貫」のパイプラインを実現するプラットフォームである。本書はCIDLSの業務要件・機能要件・非機能要件を網羅的に定義し、ステークホルダー間の共通認識を確立する。

1.2 目標（ToBe）

ID目標定量指標優先度
TB-001コンセプト入力から成果物出力まで自動化手動作業時間 90% 削減Must
TB-002シームレスドキュメント5種の自動生成全5ファイル 生成率 100%Must
TB-003CAPDkA 日次自律進化ループ稼働毎日 11:00 自動実行 稼働率 99%Must
TB-004WCAG 2.1 AA 準拠 UI全画面アクセシビリティ適合Should
TB-005販売LP経由での商用化DISTSEC.3 方法1/2 セキュリティ要件充足Should

1.3 スコープ

対象（IN スコープ）

CIDLSパイプライン6ステップ全体 / シームレス生成ドキュメント5種 / kanban・マインドマップ・STORY / LP・コンセプトスライド / CAPDkA 自律進化ループ

対象外（OUT スコープ）

外部SaaS連携（Stripe課金・認証基盤）/ 本番クラウドデプロイ / モバイルアプリ / 多言語対応（現在日本語のみ）

2. 業務要件

2.1 業務手順

Step 1: コンセプト入力

業務担当者がアイデア・課題・要望・目的・制約・現状システム・期待成果をWebフォーム/チャット/ファイルアップロードで入力する。

Step 2: コンセプト生成

AIがシステムパイプライン・アプリ画面・業務フローの3種コンセプトイメージ(PNG/HTMLスライド)を自動生成する。

Step 3: 要件定義・設計整理

ユースケース図・業務フロー図・データ設計(ER図)を自動整理し、ステークホルダーが確認・合意する。

Step 4: 統合開発・コード生成

ClaudeCowork/sonnet 4.6上でアーキテクチャ設計→コード生成→実装→テスト検証→ドキュメント生成を一貫実行する。

Step 5: 成果物出力

シームレス生成ドキュメント5種＋アプリ実行ファイル(Webアプリ/SaaS/exe/API)を一括出力する。

Step 6: 自律進化ループ

CAPDkAサイクル(C→A→P→D→K)を毎日11:00に自動実行し、反証複利で品質を継続改善する。

2.2 業務ルール

IDルール根拠

BR-002Unicode絵文字は全ドキュメント・コードで禁止AGENTS.md [G1] cp932エラー回避
BR-003フォールバック処理・ダミーデータ補完は禁止AGENTS.md [G1]

6. 業務フロー

6.1 CIDLSパイプライン メインフロー

[Step 1] コンセプト入力

INPUT: アイデア/課題/要望/目的/ゴール/業界・ドメイン/業務・制約/現状システム/期待する成果
OUTPUT: 注文票フォーマット [P1.2] 7要素

[Step 2] コンセプト生成 (GPT/Claude)

INPUT: 注文票7要素
OUTPUT: システムパイプラインコンセプトイメージ.png / アプリ画面コンセプトイメージ.png / 業務フローコンセプトイメージ.png / HTMLスライド

[Step 3] 要件定義・設計整理

INPUT: コンセプトイメージ3種
OUTPUT: ユースケース図(PNG) / 業務フロー図(PNG) / データ設計/ER図(PNG)

[Step 4] 統合開発 (ClaudeCowork/sonnet 4.6)

INPUT: 設計図・要件定義
OUTPUT: 実装コード一式 / API仕様(OpenAPI) / DBスキーマ/DDL / テストコード / 各種PNG

[Step 5] 成果物出力

INPUT: 実装コード・設計書
OUTPUT: シームレス生成ドキュメント5種(HTML) / Webアプリ / SaaS / WindowsApp(exe) / 管理画面 / API / インストーラ

[Step 6] 自律進化ループ (CAPDkA)

INPUT: 全成果物・差分・失敗ログ
OUTPUT: AGENTS.md更新 / 知識蓄積 / 次サイクル改善 (毎日11:00自動)

9. 用語定義

用語定義
CIDLSConcept-Input to Delivery-Lifecycle System。コンセプト入力からアプリ出力・自律進化まで一気通貫のパイプライン。
CAPDkACheck→ActionPlan→Plan→Do→Knowledge蓄積。CIDLS開発の永久循環サイクル。
シームレス生成ドキュメントCIDLSパイプラインが自動生成する5種のHTMLドキュメント（要求要件定義書・構成定義書兼運用保守手順書・テストシナリオ兼結果チェックリスト・ユーザーマニュアル・ドキュメント一覧）。
水平展開同期([HS])1箇所の変更後にgrep全検索→関係7箇所を完全同期する原則。
MO必須出力毎回の応答末尾に4成果物(グラフマインドマップ/カンバンボード/要求要件定義書/ソースコード)のCRUを報告すること。
devragAGENTS.md知識ベースのRAG検索ツール。v1.4.4。トークン消費をファイル全体読み込みの約40分の1に削減。
DISTSECDistribution Security。外界配置・機密流出防止・exe化境界の設計原則。
SoTSingle Source of Truth。唯一の信頼できる情報源。project_kanban.htmlがSoT。
フォールバック禁止架空データ補完・SimpleFallback・エラー隠蔽を一切禁止する原則。
反証複利各CAPDkAサイクルで過去判断を反証し、品質・知識・精度を複利的に向上させること。
