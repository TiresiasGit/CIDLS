## 水平展開同期原則 [HS]

```yaml
根本原則: 共通する内容は全く同じ観点で関係するすべてに水平展開する
  同一概念 → 同一表現 → 同一適用 (部分適用・観点ズレ禁止)
  適用範囲: コード / 原則 / 用語 / ルール / 設定 / ドキュメント / テスト

手順:
  1. 1箇所変更
  2. grep全検索
  3. 以下7箇所に完全同期
  4. 旧名残存ゼロ確認

対象7箇所:
  1. 定義 (クラス/関数/定数の定義箇所)
  2. 辞書 (dict・設定値での参照箇所)
  3. 設定 (config・settings・ini)
  4. 分岐 (if/elif/match/case)
  5. import (import文・from import)
  6. ドキュメント (docstring・コメント・README)
  7. テスト (test_*.py)
```

### 実行コマンド
```bash
# Step 1: 全検索
grep -r "変更する概念名" . --include="*.py" --include="*.md" -n

# Step 2: 旧名残存ゼロ確認
grep -r "旧名称" . --include="*.py" -n
# → 結果がゼロであること
```

### 禁止パターン
- 部分適用: 一部だけ変更して他を放置
- 観点ズレ: 関連するが微妙に違う概念の片側だけ修正

## 呼び出し元意識の修正統合 [T1.6]

### 4Phase手順
```yaml
Phase1.呼び出し関係完全調査:
  - grep -r "func_name" .
  - メソッド継承確認
  - import参照確認
  - getattr動的呼び出し確認

Phase2.影響範囲評価:
  - 影響する全呼び出し側を特定
  - API契約変更 (引数・戻り値・例外)
  - 下位互換性確認
  - テスト影響確認

Phase3.統合的修正:
  - 修正対象 + 全呼び出し側を同期
  - IF変更時は段階的移行
  - エラーハンドリングを統一

Phase4.整合性検証:
  - 全呼び出しパターンの動作確認
  - 型整合性確認
  - エッジケース・例外確認
```

## パラメータ整合性 [T2.4]

同一概念は同一値を使う。異なる値は理由をコメントで明示。

```python
# 禁止: 同一概念に異なる値
WINDOW_SIZE_SSA = 10    # SSA分解用
WINDOW_SIZE_ML = 15     # ML学習用 ← 同じ概念で値が違う

# OK: 統一 or 理由明示
WINDOW_SIZE = 10  # 全体共通

# OK: 異なる理由が明確
WINDOW_SIZE_SSA = 10   # SSA分解: 短期パターン捕捉
WINDOW_SIZE_ML = 15    # ML学習: 長期依存関係考慮 (理由明示)
```

チェック手順:
1. `grep -r "定数名" .` で全箇所検索
2. 値を比較
3. 不一致は数学的に妥当か検証
4. 不一致理由をコメントで明示
