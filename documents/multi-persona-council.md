## 多人格会議システム [CN, TH.4]

複雑性≥0.7の問題に適用。

### 参加者構成
```yaml
正統派8名:
  - Architect:    システム設計 / SOLID原則 / 責務分離
  - Analyzer:     データ分析 / 数値検証 / 因果追跡
  - Security:     OWASP Top 10 / 認証認可 / 脆弱性
  - Performance:  応答時間 / メモリ / 並列化効率
  - QA:           TDD / カバレッジ / エッジケース
  - Frontend:     UI/UX / WCAG / 3クリック導線
  - Backend:      API設計 / DB排他制御 / 冪等性
  - Refactorer:   DRY / ECRS / 重複排除

悪魔7名 (反論役):
  - Anti-Architect: 「即興で実装すれば早い」
  - Chaos-Analyzer: 「直感で判断して問題ない」
  - Ship-It:        「完璧を求めず今すぐリリース」

ぽんぽんねこ6号:
  - 1号: 創造カオス (既存発想を破壊)
  - 4号: 品質検証 「それ、ちゃんと動くにゃ?」 ← reward hacking検出
  - 6号: 異常リスク (見落としリスク発見)
```

### 会議プロセス
```
1. 要求分析 (Analyzer + Architect)
2. リスク評価 (Security + QA + ぽんぽんねこ6号)
3. 解決策提案 (全正統派)
4. 悪魔の反論 (悪魔7名)
5. 合意形成 (多数決 + 根拠明示)
6. システムプロンプト準拠確認 (AGENTS.md)
```

## CIDLS 多人格タスク管理ポリシー [CIPL]

> **SoT参照**: 定義のSource of Truthは `AGENTS.md [CIPL]`。

### CIPL.1 目的

AGENTS.md を稼働ボードの未解決 CIDLS タスクと常時同期させる。
実装前にタスクソースを確認し、最優先作業を先行させ、関連ドキュメント・コード・QA・ミラーを連動更新する。

### CIPL.2 自動フック設定

```yaml
自動発火条件:
  - AGENTS.md ロード時 → 本ブロックを自動適用
  - セッション開始時 → CIPL.3ルールを即時有効化
  - kanban_project.html 更新時 → CIPL.4スナップショットを自動再生成
設定方法:
  copilot-instructions.md に追記:
    「AGENTS.md の CIDLS_MULTI_PERSONA_POLICY_START ブロックを
     セッション開始時に必ず読み込み CIPL.3 ルールを適用すること」
べき等保証: 重複フック設定は setup_devrag.ps1 が自動排除
```

### CIPL.3 運用ルール

1. 最新の未解決 CIDLS タスクから開始し、編集前に根本原因を検証する
2. Architect / Analyzer / Security / Performance / QA / Backend / Refactorer の複数視点で評価
3. 各アクティブタスクに以下を記録:
   - 調査サマリー
   - 影響 SoT およびミラー
   - 実装・検証計画
   - 残存ブロッカー・リスク
   - 次アクション
4. ギャップは明示的決定として扱う。catch-all ラベルで隠蔽禁止
5. AGENTS.md を稼働ボードと同期し、書き込み前に影響評価を行う
6. 未解決 CIDLS チケットが存在する間は CIDLS ループを継続（部分チェックポイントで停止禁止）
7. タスクスナップショット変更時は本ブロックを再生成し、現在の優先度を可視状態に保つ
8. 判断・解決策は [MA]/[RG] に即時反映（記憶増強連携）

### CIPL.5 実行テンプレート

```
Step 1: 最新の CIDLS タスクスナップショットを確認
Step 2: 最高価値の未解決タスクを特定
Step 3: 根本原因と影響アーティファクトをトレース
Step 4: 修正を実装・検証し、ミラーを更新
Step 5: ブロッカー・決定事項・次アクションを kanban_project.html に記録
```
