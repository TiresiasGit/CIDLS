# 差分切り分け知識ベース (Diff Isolation) [DIFF]

> **SoT参照**: 定義のSource of Truthは `AGENTS.md [DIFF]`。
> このファイルはdevrag RAG用実装例・コードパターンを収録。
> [PORT] エクスポート/インポートの差分切り分けも同一単純 (Before/Afterスキーマバージョンを情報に含める)。

AGENTS.mdの [DIFF] セクションに基づく、差分切り分け原則の詳細知識ベース。

## 核心概念 [DIFF.1]

差分切り分け := 「変わった部分」と「変わっていない部分」を完全に識別してから行動する

### なぜ差分切り分けが必要か
- 「全体的に修正した」= 根本原因が特定できない最悪の状態
- 差分を曖昧にすると副作用・regression・reward hackingが検出不能になる
- 差分切り分けにより、バグ特定速度が平均3倍に向上する

### 禁止事項
- 「全体的に整理した」という曖昧な変更理由
- 差分報告なしの「修正完了」宣言
- Before確認なしの修正着手
- 前後比較なしのパフォーマンス向上主張
- 複数の無関係な修正を1回の「変更」として混在させる

## 差分切り分け5ステップ [DIFF.2]

### Step 1: Before記録
grep/Read で変更前の状態を完全記録。行番号・内容・依存関係をログ出力。
```powershell
grep -n "対象パターン" ファイル名
```

### Step 2: 変更箇所特定 (バイナリサーチ的アプローチ)
最小単位で変更箇所を特定。「絶対に変えてはいけない箇所」をリスト化。
変更箇所をコメントで明示: `# [DIFF] 変更: 理由`

### Step 3: After記録
変更後を同条件でgrep記録。Step1との差分を行番号付きで抽出。

### Step 4: 差分比較・明示
形式: `ファイル名:行番号 | Before: <旧値> | After: <新値> | 理由: <変更理由>`

### Step 5: 副作用チェック
変更した概念をgrep全検索。テスト実行で変更前後のPass/Fail変化を確認。
意図しない変化 → 即座にロールバック判断。

## 差分報告フォーマット [DIFF.3]

```
[DIFF] 変更報告
  変更ファイル:     <ファイルパス>
  変更行数:         追加X行, 削除Y行, 修正Z行
  Before:           <変更前コード/設定値/状態>
  After:            <変更後コード/設定値/状態>
  変更理由:         <Why1回答 - 根本原因との接続>
  影響範囲:         <grep結果で他箇所への波及を明示>
  副作用:           なし (grep確認済み) | あり → <具体的影響先>
  ロールバック手順: <復元方法>
```

## 適用トリガー [DIFF.4]

必須適用場面:
- バグ修正時: バグ再現条件と修正差分を1:1対応させる
- リファクタリング時: 動作変化ゼロを差分で証明
- 設定変更時: Before/After値と影響測定値を明示
- 依存ライブラリ更新時: 影響する関数シグネチャ変化を列挙
- 水平展開 [HS] 実行時: 7箇所の変更を各々個別に差分報告

## バイナリサーチ的デバッグ [DIFF.5]

問題: どこが原因か不明な場合のO(log N)特定手順
1. 全体をA/B2分割し、問題がどちら側にあるか確認
2. 問題側をさらに2分割
3. 繰り返し → O(log N)で特定
4. 特定できたら [DIFF.2] Step1-5を実行

適用例:
- N個のコミットでどれがバグ導入か → git bisect的アプローチ
- N行の設定ファイルでどれが問題か → コメントアウトで2分探索
- N個のテストでどれが副作用を起こすか → テスト分割実行

## CAPDkAサイクルとの連携

| フェーズ | 差分切り分けアクション |
|---------|----------------------|
| C(Check) | 現状(Before)をgrepでスナップショット記録 |
| A(ActionPlan) | 変更箇所を最小単位で特定し影響範囲をリスト化 |
| D(Do) | 各実装ステップで差分報告フォーマットを適用 |
| kA(Knowledge) | 差分報告をkanban_project.htmlに蓄積 |

## 水平展開 [HS] との協調

1変更 → 差分報告[DIFF.3]作成 → grep全検索 → 7箇所に波及するか確認
→ 波及あり → 各箇所に個別の差分報告[DIFF.3]を作成
→ 全7箇所の差分報告を束ねてkAへ蓄積
