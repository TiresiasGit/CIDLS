4. データ仕様

4.1 DA表エントリ構造

DA表 (診断分析表) は graph_project_mindmap.html の HTML テーブルとして管理される。 1行 = 1件のアセスメント観測事項。

フィールド型説明例
DA-IDstring連番ID (DA-001〜)DA-012
分類enum F/A/U事実(F) / 想定(A) / 未知(U)F
軸string[CS.2]5軸のいずれか軸1 機能維持
内容text[DONE/TODO/BLOCKED YYYY-MM-DD] 詳細説明[DONE 2026-05-12] ...
出典/根拠stringファイル名・コマンド・スクリプト名CAPDkAサイクル実行 2026-05-12 第8回
優先度enum P1/P2/P3P1=緊急 P2=重要 P3=中期P1

4.2 成果物マスターデータ構造

フィールド型制約
ファイル名string日本語可 / .html / 重複禁止
カテゴリenumCIDLSコア / SW納品 / スクリプト
バージョンsemvervX.Y.Z 形式
ステータスenumDONE / TODO / BLOCKED / IN_PROGRESS
優先度enumP1 / P2 / P3
生成日dateYYYY-MM-DD 形式

4.3 DuckDB / Parquet データ規約

将来的にDuckDB/Parquetを使用するデータパイプラインを構築する場合の規約を定める。

規約項目内容
DB パスPath(os.environ.get("DB_PATH", "data/local.duckdb"))
ファイル形式Parquet (snappy圧縮) / 目標圧縮率 ≥30%削減
スキーマバージョンschema_version / exported_at / record_count を必ず付加
round-tripexport → import → 同一結果 (自動テスト必須)
NULL許可import前にサーバーサイドで検証必須 (クライアント申告禁止)

5.2 セキュリティ (AGENTS.md [SEC] / OWASP Top 10)

区分要件実装箇所
[SEC.BILLING_ENFORCE]課金はサーバーサイドのみで決定将来 Stripe 統合時に適用
[SEC.PII_AUTH_GATE]未認証状態でPII返却禁止将来 Web公開時に適用
[DISTSEC.2]Secret を配布物・public・HTML に含めない.gitignore / 配布ZIP確認
絵文字禁止Unicode絵文字は cp932 エラーの原因 → 禁止全 .py / .ps1 / .bat
pip 禁止uv 専用。pip コマンドは使用禁止全 Python スクリプト
入力検証コマンドライン引数はargparseで型・必須を強制generate_*.py
